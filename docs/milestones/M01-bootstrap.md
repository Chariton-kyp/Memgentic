## Phase 1.6: End-to-End Verification

**Goal:** Verify the full pipeline works with real data.

### Tasks

1. **Check if Ollama is running locally:**
   ```bash
   curl http://localhost:11434/api/tags
   ```
   If not running, start it. If the Qwen3-4B model isn't pulled:
   ```bash
   ollama pull qwen3-embedding:4b
   ```

2. **Test the embedding pipeline:**
   ```python
   # Quick script to test embedding generation
   uv run python -c "
   import asyncio
   from memgentic.processing.embedder import Embedder
   from memgentic.config import settings

   async def test():
       e = Embedder(settings)
       vec = await e.embed('Hello, this is a test memory about Python.')
       print(f'Embedding dimensions: {len(vec)}')
       assert len(vec) == 768, f'Expected 768, got {len(vec)}'
       print('Embedding OK!')

   asyncio.run(test())
   "
   ```

3. **Test the full ingestion pipeline:**
   ```bash
   uv run mneme remember "Test memory: Memgentic uses Qdrant for vector search and SQLite for metadata" --topics "mneme,qdrant,sqlite" --source claude_code
   ```

4. **Test search:**
   ```bash
   uv run mneme search "vector database"
   ```

5. **Test source listing:**
   ```bash
   uv run mneme sources
   ```

6. **Import real Claude Code conversations** (if they exist at `~/.claude/projects/`):
   ```bash
   uv run memgentic import-existing
   ```

7. **Test MCP server startup:**
   ```bash
   # Test stdio transport starts without error (will hang waiting for input, Ctrl+C)
   timeout 5 uv run mneme serve || true
   ```

### Acceptance Criteria
- [ ] Embeddings generate correctly (768 dimensions)
- [ ] Can remember and search a memory
- [ ] Sources command shows stored memories
- [ ] Import processes real conversation files (if available)
- [ ] MCP server starts without error
